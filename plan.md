# bestcityinchina.site SEO 详细优化计划

> 版本：v1.0  
> 生成日期：2026-02-20  
> 适用周期：2026-02-21 至 2026-04-03（6 周）

## 1. 目标与范围

### 1.1 业务目标
- 提升自然搜索可见性与可索引质量，避免索引污染。
- 提升首页性能指标，改善核心词排名潜力。
- 完成多语言 SEO 信号一致性，提升国际化页面收录稳定性。
- 提升搜索结果点击率（CTR）与内容可信度（E-E-A-T）。

### 1.2 本计划覆盖范围
- 技术 SEO：抓取、索引、规范化、状态码、robots、sitemap。
- 页面 SEO：title、description、hreflang、canonical、语言标签。
- 性能 SEO：首页 Web Vitals、关键渲染路径、静态资源缓存。
- 内容 SEO：元信息本地化、信息增益、内链与主题集群。

### 1.3 当前基线（审计结果）
- 软 404：存在（随机不存在 URL 返回 `200 + index,follow`）。
- 多语言首页信号冲突：`/zh`、`/ja`、`/ko` 的源码 canonical 指向 `/en`，且 `lang` 为 `en`。
- robots 与索引策略冲突：`/quiz`、`/result` 被 robots 禁抓，但页面为 `index,follow`。
- 首页性能：Lighthouse 约 `56-57`，LCP 约 `10.5s~11.8s`（移动端）。
- 多语言描述偏短：多页 description 明显不足。

## 2. 北极星 KPI（按阶段验收）

| 指标 | 当前基线 | 目标 | 截止日期 |
| --- | --- | --- | --- |
| 软 404 URL 数 | >0 | 0 | 2026-02-27 |
| 非法路径 HTTP 状态 | 200 | 404/410 | 2026-02-27 |
| 多语言首页 canonical/lang 冲突数 | 3 | 0 | 2026-02-27 |

## 3. 执行原则

- 先索引健康，再做性能，再做 CTR 放大。
- 先修冲突信号，再扩大内容规模。
- 所有改动必须“可回滚、可监控、可验收”。

## 4. 分阶段优化计划

## P0（最高优先级）：索引与抓取治理
**时间：2026-02-21 ~ 2026-02-27**

### P0-1 修复软 404（必须）
**问题**：不存在路径返回 200 且可索引。  
**动作**：
- 路由层处理未知路径，返回 `404`（或明确下线页返回 `410`）。
- 404 模板增加 `<meta name="robots" content="noindex,follow">`。
- 404 页面不出现在 sitemap。
- 不允许“未知 URL 自动回退到首页且 200”。

**验收**：
- `curl -I https://bestcityinchina.site/en/non-existent-page-xyz` 返回 404/410。
- 10 个随机不存在 URL 均非 200。
- GSC 软404趋势在 1-2 周内下降。

### P0-2 统一多语言首页 SEO 信号
**问题**：`/zh`、`/ja`、`/ko` 源码信号偏英文。  
**动作**（推荐采用“真多语言”方案）：
- `/zh`：`<html lang="zh-CN">`，canonical 自指 `/zh`，title/description 中文化。
- `/ja`：`lang="ja"`，canonical 自指 `/ja`，title/description 日文化。
- `/ko`：`lang="ko"`，canonical 自指 `/ko`，title/description 韩文化。
- 保证 hreflang 集群互相可回指，且 `x-default -> /en`（或自定义默认语）。

**备选方案（若不做多语言首页）**：
- `/zh` `/ja` `/ko` 永久 301 到 `/en`；
- 同步移除对应 hreflang 与 sitemap 入口，避免信号矛盾。

**验收**：
- 4 个首页 canonical 与语言匹配且无跨语种冲突。
- 抓取源码与渲染后结果一致。

### P0-3 处理 robots 与 noindex 冲突（quiz/result）
**问题**：`robots.txt` Disallow 了 quiz/result，但页面本身可索引。  
**动作**：
- 若目标是不收录：
  - 从 `robots.txt` 移除 Disallow（允许抓取）；
  - 在 quiz/result 页面加 `meta robots=noindex,follow`；
  - canonical 指向自身（或不输出 canonical，避免误导到首页）。
- 若目标是收录：
  - 保留 index,follow；
  - 移除 robots Disallow；
  - 为页面提供唯一 title/description 与可索引内容。

**验收**：
- 无“blocked by robots 但仍被索引”类型告警。
- quiz/result 策略在 robots、meta、canonical 三处一致。

### P0-4 sitemap 与状态码一致性
**动作**：
- sitemap 仅保留 `200 + indexable + canonical self` 页面。
- 排除 404/noindex/重定向 URL。
- 保留 `lastmod`，并确保与真实更新时间一致。

**验收**：
- sitemap URL 抽检 100% 满足收录条件。
- GSC sitemap 报告无大规模异常。

## P1（高优先级）：首页性能与渲染路径优化
**时间：2026-02-28 ~ 2026-03-13**

### P1-1 字体策略重构（核心）
**问题**：首页阻塞式加载多语种多字重 Google Fonts。  
**动作**：
- 按语言拆分字体集合，避免首页一次性加载所有语种字体。
- 将字重缩减到必要档位（例如 400/600）。
- 采用本地托管 `woff2` + `unicode-range` 子集化。
- 非关键字体 `font-display: swap`。
- 首屏只保留 1 组必要字体，其他延迟。

**验收**：
- 字体阻塞时间显著下降。
- 首页 LCP 降至 <2.5s（移动端）。

### P1-2 CSS/JS 体积与关键路径
**动作**：
- 将首页与内容页样式拆分，避免首页加载内容页冗余样式。
- 清理未使用 CSS/JS，优先优化首页 bundle。
- 关键 CSS 内联（仅首屏必要部分），非关键 CSS 延后加载。
- 非关键脚本（广告、统计）延迟到首屏可交互后再执行（不影响合规前提下）。

**验收**：
- 首页 Lighthouse Performance >=80（阶段目标）。
- 首页 Speed Index 显著下降。

### P1-3 缓存策略升级
**动作**：
- 对 hash 静态资源设置：`cache-control: public, max-age=31536000, immutable`。
- HTML 保持较短缓存 + revalidate。

**验收**：
- 静态资源重复访问命中缓存。
- 资源头与文件指纹策略一致。

## P2（中优先级）：页面信息增益与 CTR 优化
**时间：2026-03-07 ~ 2026-03-20**

### P2-1 标题与描述本地化改写
**动作**：
- 为每个语言版本独立写 title/description，避免“跨语种同模板”。
- 长度建议：
  - 英文 title 50-60 字符，description 120-155 字符；
  - 中文 description 建议 45-80 字；
  - 日/韩文 description 结合语义自然扩展，避免过短。
- 每页包含 1 个主意图词 + 1 个差异化卖点（如“18 问题匹配”）。

**验收**：
- 过短 description 页面数降至 <=2。
- 主要入口页 title/description 唯一化 100%。

### P2-2 OG/Twitter 多语言对齐
**动作**：
- OG title/description 与页面语言一致。
- 保持 `og:url` 与 canonical 一致。
- 语言页可补充 `og:locale`（如 `zh_CN`, `ja_JP`, `ko_KR`）。

**验收**：
- 抽检页面社交卡片字段与页面语言一致。

### P2-3 结构化数据增强
**动作**：
- 保留已有 WebSite/Article/Breadcrumb。
- Article 增补作者实体信息（作者页 URL、组织信息一致性）。
- 确保 `dateModified` 真实更新，避免长期不变。

**验收**：
- Rich Results / Lighthouse SEO 无结构化错误。

## P3（持续优化）：内容质量与主题权威
**时间：2026-03-14 ~ 2026-04-03（并持续）**

### P3-1 构建主题集群与内链
**动作**：
- 以“首次来华城市选择”为核心主题，扩展子主题：
  - 城市对比（北京/上海/成都等）
  - 预算、天数、季节、签证与支付准备
  - 不同旅行风格决策路径
- 每篇文章新增 3-5 条上下游内链，锚文本语义化。

**验收**：
- 所有指南页存在明确上下游内链链路。
- 平均每页有效内链数提升。

### P3-2 E-E-A-T 信号增强
**动作**：
- 增加作者与编辑规范页面（Who wrote/reviewed/updated）。
- 明确“最后更新日期 + 更新摘要”。
- 引用权威来源并做外链出处标注。

**验收**：
- 文章页具备作者、发布时间、更新信息、来源信息。

### P3-3 稳定迭代节奏
**动作**：
- 每周发布/更新至少 2 篇高价值内容。
- 每两周做一次内容刷新（旧文更新与内链补强）。

**验收**：
- 覆盖词库持续增加，长尾页面收录提升。

## 8. 风险与预案

- 风险 1：SPA/SSR 渲染链路复杂，源码与渲染结果不一致。  
  预案：上线前同时做“源码抓取 + 浏览器渲染”双重校验。

- 风险 2：广告/统计脚本延迟影响业务统计口径。  
  预案：先灰度 10% 流量，确认埋点无明显丢失再全量。

- 风险 3：多语言文案产能不足，导致改动滞后。  
  预案：先覆盖首页+核心指南页，再逐步扩展次级页面。

- 风险 4：索引策略调整后短期波动。  
  预案：GSC 提交验证并观察 2-4 周，避免频繁来回改策略。
